# 本文生成プロンプト（M4-2）

痛み仮説を1文織り込んだ、その会社にしか出せない個別最適化メール本文を生成する。テンプレ丸出しは禁止。

## 入力
- 会社名: {{company_name}}
- 事業内容要約: {{business_summary}}
- 痛み仮説: {{pain_hypothesis}}
- 送信者: {{sender_name}}

## 指示
1. 痛み仮説を1文織り込み、「なぜこの会社に送っているのか」が伝わる書き出しにする。
2. デジタル化・AI導入補助金など「予算がつく前提の提案」であることに触れる（補助金名を断定しすぎず、「対象になり得る」程度のトーンにする）。
3. 売り込み色を抑え、まずは情報交換・15〜30分程度の壁打ちを打診する着地にする。
4. 本文のみを出力する（件名・署名は含めない）。文字数は300〜500字程度。
5. 誇大表現・過度な緊急性の煽り・実績の水増しは禁止。

## 出力形式
以下のJSON形式のみで回答する（説明文・コードブロック記号は不要）。

```
{
  "subject": "件名（20字前後、痛み仮説に紐づく具体的な件名）",
  "body": "本文（署名を含まない）"
}
```
